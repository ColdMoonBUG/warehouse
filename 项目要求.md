# 食品中间商进销存/车销系统 — 前端后台（一期：库存闭环）

## TL;DR
> **Summary**: 根据目前现有的ruoyi框架大修大改实现以下项目，优先跑通“主仓↔业务员车库”的库存闭环（入库、调拨、库存实时查看），地图用免费 OSM + Leaflet 并支持地址搜索选点。
> **Deliverables**:
> - Vue3 管理台工程骨架（PC为主 + 简单移动适配）
> - 基础资料：厂家/商品/门店/员工 CRUD
> - 库存总览：主仓库存 + 按业务员车库库存
> - 单据：入库单、调拨单（保存草稿 + 过账 + 作废反冲）
> - 首页：地图打点 + 门店选点（Leaflet + OSM）+ 地址搜索（Nominatim）
> - 全部使用 mock 数据（localStorage 持久化），后端后续按前端契约补齐
> **Effort**: Medium
> **Parallel**: YES - 3 waves
> **Critical Path**: 工程脚手架 → Layout/路由 → Mock数据层 → 基础资料 → 库存总览 → 入库/调拨单据过账与反冲 → 地图打点/选点

- 地图优先免费方案：OSM 底图；仍需地址搜索定位

## Work Objectives
### Core Objective
用最少页面/最少交互，确保用户随时能看到：
- 主仓每个商品剩余数量
- 每个业务员车库（车上）每个商品剩余数量
且这些数量能被“入库单、调拨单”的过账/作废准确驱动。

### Deliverables
1. `apps/admin-web` 前端工程（Vue3+Vite+TS+ElementPlus）
2. 管理台 Layout（菜单、面包屑、页面容器、移动折叠菜单）
3. Mock 数据层（localStorage 持久化）+ 统一 API 适配层
4. 页面：
   - 首页看板：Leaflet 地图、门店打点、门店选点录经纬度、地址搜索
   - 基础资料：厂家/商品/门店/员工 CRUD
   - 库存总览：主仓库存 + 业务员车库库存
   - 入库单：草稿/过账/作废（反冲）
   - 调拨单：草稿/过账/作废（反冲），支持主仓→车库、车库↔车库

### Definition of Done (agent-verifiable)
- [ ] 前端可启动：`pnpm -C apps/admin-web install && pnpm -C apps/admin-web dev` 正常运行
- [ ] 前端可构建：`pnpm -C apps/admin-web build` 退出码 0
- [ ] Playwright 自动化通过：完成“建档→入库→调拨→库存变化→作废反冲→库存恢复”全链路
- [ ] 地图页自动化通过：地图成功渲染；门店打点出现；地址搜索返回候选；选点后经纬度写入表单

### Must Have
- 库存展示必须以“仓库(主仓/车库) + 商品”为维度
- 单据必须有 `draft/posted/voided`（作废）状态
- 过账才影响库存；作废必须反冲库存（生成反向流水）
- 所有录入页必须“表单+明细表格”，支持回车/快速新增行（不要求极致键盘流，但不能难用）

### Must NOT Have (guardrails)
- 不实现登录/权限
- 不实现销售单、门店退单、应收回款、提成报表（二期）
- 不引入复杂采购/财务总账概念
- 不做批次库存管理（你的业务明确不需要）

    - 生成反向库存流水并回滚库存

  **Acceptance Criteria**:
  - [ ] 过账后库存增加
  - [ ] 作废后库存回到过账前
